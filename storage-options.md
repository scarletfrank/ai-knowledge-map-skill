# 通用知识地图存储方案

## 方案对比

| 特性 | Feishu方案 | Obsidian方案 |
|------|------------|--------------|
| **存储位置** | 云端（飞书文档） | 本地（Markdown文件） |
| **协作能力** | ✅ 多人实时协作 | ❌ 个人使用为主 |
| **离线访问** | ❌ 需要网络 | ✅ 完全离线可用 |
| **版本控制** | ❌ 有限历史版本 | ✅ Git完整版本控制 |
| **移动端** | ✅ 官方App支持 | ✅ 第三方App支持 |
| **自定义程度** | ⚠️ 有限定制 | ✅ 高度可定制 |
| **数据所有权** | ⚠️ 依赖飞书 | ✅ 完全自主控制 |
| **API集成** | ✅ 飞书Open API | ✅ 文件系统操作 |

## Feishu方案详情

### 核心能力
- 利用 `feishu-docs-v2` 技能进行文档读写
- 支持块级操作（创建、更新、删除特定内容块）
- 自动同步到飞书知识库
- 支持团队协作和评论

### 技术要求
- 需要配置飞书应用权限
- 依赖飞书Open API
- 需要稳定的网络连接

### 适用场景
- 团队知识共享
- 需要移动端便捷访问
- 偏好云端存储和自动备份

## Obsidian方案详情

### 核心能力
- 本地Markdown文件存储
- 双向链接和知识图谱可视化
- 模板系统和标签分类
- Git版本控制集成

### 技术要求
- 需要安装Obsidian应用
- 本地存储空间
- 可选：GitHub同步配置

### 适用场景
- 个人知识管理
- 离线工作环境
- 需要高度自定义和数据控制
- 开发者/技术用户偏好

## 选择建议

### 选择Feishu方案如果：
- 你已经在使用飞书进行工作
- 需要与团队成员共享知识
- 希望在手机上随时查看和编辑
- 不想管理本地文件和版本控制

### 选择Obsidian方案如果：
- 你重视数据隐私和所有权
- 经常在离线环境下工作
- 喜欢高度自定义的工作流
- 希望利用Git进行版本控制和备份
- 是技术用户，熟悉Markdown格式

## 混合使用
你也可以同时使用两种方案：
- 日常记录使用Obsidian（离线优先）
- 重要知识定期同步到Feishu（团队共享）
- 通过自动化脚本实现双向同步
